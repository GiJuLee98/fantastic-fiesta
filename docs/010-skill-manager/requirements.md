# 010-skill-manager: requirements

1. 개요 및 목적 [1]
    - `skill-manager`는 사용자와 상호작용하여 이 프로젝트에 등록된 skill들을 종합적으로 관리(조회, 추가, 수정, 삭제)해주는 스킬이다. [1-1]
    - 신규 스킬 추가, 기존 스킬 구성요소 수정 및 삭제에 따른 의사결정 프로세스를 챗봇 형태로 자동화하여, 대화를 통해 정밀한 작업을 수행할 수 있도록 돕는다. [1-2]
    - `docs/008-gemini-skill-guide/gemini-skill-addition-guide.md` 및 `docs/009-skill-template/gemini-skill-template.md`에 정의된 명세 및 가이드를 준수하여 관리 작업을 수행한다. [1-3]

2. 기능 요구사항 [2]
    - 기존 스킬 정보 조회 및 목록 제공 [2-1]
        - 프로젝트 내 `.gemini/skills/` 하위에 존재하는 기존 스킬들의 목록을 조회하고, 각 스킬의 명세(`SKILL.md`) 요약을 제공한다. [2-1-1]
    - 신규 스킬 추가 (Create) [2-2]
        - 스킬 식별자, 명칭, 용도 및 개별 도구(Tools)의 세부 명세에 대해 인터랙티브 질문을 통해 정보를 수집한다. [2-2-1]
        - 애매하거나 복잡한 항목은 예상되는 모범 예시를 제시하여 선택을 편하게 돕는다. [2-2-2]
        - 수집된 의사결정 결과를 기반으로 폴더 생성, `SKILL.md`, `config.json`, 실행 스크립트(`scripts/run.ps1`)를 자동 생성한다. [2-2-3]
        - 루트 `.gemini/config.json`의 `enabled_skills`에 새 스킬을 자동으로 추가한다. [2-2-4]
        - **[신규]** 대화 내용 및 의사결정 배경을 담은 히스토리 문서(`HISTORY.md`)를 `.gemini/skills/[skill-name]/HISTORY.md`에 새로 작성한다. [2-2-5]
    - 기존 스킬 수정 (Update) [2-3]
        - 수정하고자 하는 타겟 스킬을 지정받고, 대화를 통해 수정할 대상(도구 명세 추가/삭제/수정, 설명 업데이트 등)을 명확하게 파악한다. [2-3-1]
        - 사용자의 요구사항을 토대로 `SKILL.md`, `config.json`, 혹은 관련 스크립트 파일을 안전하게 업데이트한다. [2-3-2]
        - 애매한 수정 사항의 경우, 수정 예상 변경점(Diff 또는 대안 예시)을 사용자에게 사전 제시하고 승인을 묻는다. [2-3-3]
        - **[신규]** 대화 내용과 수정 배경 요구사항을 기존 `.gemini/skills/[skill-name]/HISTORY.md` 파일에 날짜 및 변경 유형별로 누적(Append)하여 추가한다. [2-3-4]
    - 기존 스킬 삭제 (Delete) [2-4]
        - 사용자가 요청한 스킬을 프로젝트에서 제거한다. [2-4-1]
        - 스킬 폴더를 물리적으로 삭제하기 전, 정말 삭제할 것인지 그리고 루트 `.gemini/config.json`에서 제외할 것인지 대화를 통해 명확하게 최종 컨펌을 받는다. [2-4-2]
        - 승인 시, 루트 `.gemini/config.json`에서 해당 스킬을 제거하고 폴더를 안전하게 삭제한다. [2-4-3]

3. 비기능 및 환경 요구사항 [3]
    - 스킬의 폴더명 및 식별자는 영문 소문자와 하이픈(`-`)만을 사용하도록 강제 검증을 적용해야 한다. [3-1]
    - 스킬의 수정 및 삭제 등 변경 작업 수행 전에는 반드시 원본 데이터 유실을 방지하기 위한 안전장치를 마련해야 한다. [3-2]
    - 에이전트의 작동 환경(Windows)에 맞는 파일 시스템 제어 및 스크립트 처리가 가능해야 한다. [3-3]
    - `HISTORY.md`는 에이전트 실행에는 영향을 미치지 않도록 관리 목적용 문서 포맷으로 관리한다. [3-4]
