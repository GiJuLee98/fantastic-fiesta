# ImGui 프로젝트 계획서

## 1. 프로젝트 개요 [1]

### 1.1. 핵심 목표 및 문제 정의 (tas:1-1) [1-1]
- 본 프로젝트는 Windows OS 환경에서 C++ 및 OpenGL, GLFW를 기반으로 한 고성능 데스크톱 GUI 애플리케이션의 구조적 기틀을 마련하는 것을 목표로 한다. [1-1-1]
- 기존 GUI 개발 프레임워크들의 복잡한 이벤트 루프 및 무거운 런타임 오버헤드 문제를 해결하기 위해, 즉시 모드(Immediate Mode) GUI 라이브러리인 Dear ImGui를 도입하여 직관적이고 효율적인 렌더링 파이프라인을 구축한다. [1-1-2]
- 개발 생산성 저하와 창 관리의 제약을 해결하기 위해 개발자 친화적인 디버깅 및 도구 개발이 가능한 다중 뷰포트(Multi-viewports) 및 도킹(Docking) 인터페이스를 제공한다. [1-1-3]

### 1.2. 적용 범위 및 사용자 타겟층 (tas:1-2) [1-2]
- 본 프로젝트의 범위는 GLFW 창 관리 라이브러리와 OpenGL 3.3 이상의 그래픽 API를 연동하여 Dear ImGui GUI 애플리케이션의 핵심 골격을 형성하는 데 한정한다. [1-2-1]
- 대상 사용자는 C++ 기반의 임베디드 도구 개발자, 게임 툴 프로그래머, 그리고 가볍고 빠른 데스크톱 유틸리티를 필요로 하는 시스템 엔지니어이다. [1-2-2]
- 다중 작업 공간과 모니터 확장이 필수적인 전문가 개발 환경을 지원하며, 윈도우 경계를 넘나드는 자유로운 드래그 앤 드롭 UI를 제공한다. [1-2-3]

## 2. 개발 환경 및 기술 스택 [2]

### 2.1. 개발 환경 규격 (tas:2-1) [2-1]
- 대상 운영체제는 Windows 10/11 64비트 환경을 기준으로 개발 및 배포를 수행한다. [2-1-1]
- 통합 개발 환경(IDE)은 Visual Studio 2022를 기본으로 채택하고, MSVC 컴파일러를 사용하여 빌드한다. [2-1-2]
- 빌드 시스템은 CMake 3.20 이상을 사용하여 플랫폼 독립적인 프로젝트 구성 환경을 제공하며, 의존성 관리가 가능하도록 구성한다. [2-1-3]

### 2.2. 기술 스택 및 라이브러리 연동 규격 (tas:2-2) [2-2]
- 개발 언어는 C++17 표준 규격을 기반으로 하여 작성하되, 필요 시 C++20 표준 컴파일이 가능하도록 프로젝트 설정을 유연하게 유지한다. [2-2-1]
- 그래픽 API는 현대적인 OpenGL 3.3 Core Profile 이상을 타겟팅하여 GPU 가속 렌더링을 구현한다. [2-2-2]
- OS 윈도우 생성 및 입력 처리를 위해 GLFW 3.3 이상 라이브러리를 사용하며, OpenGL 컨텍스트 생성 및 입출력 이벤트를 Dear ImGui 백엔드와 연결한다. [2-2-3]

## 3. 기능 사양 및 마일스톤 계획 [3]

### 3.1. GUI 핵심 기능 규격 (tas:3-1) [3-1]
- 사용자가 개별 도구 창을 메인 윈도우 내부에서 결합하고 분리할 수 있는 도킹(Docking) 아키텍처를 활성화한다. [3-1-1]
- 메인 OS 윈도우 외부로 ImGui 창을 드래그하면 새로운 독립 OS 윈도우로 분리되어 표시되는 멀티 뷰포트(Multi-viewports) 기능을 지원한다. [3-1-2]
- 사용자 정의 레이아웃을 프로그램 종료 후에도 유지할 수 있도록 ini 설정 파일 자동 저장 및 로드 매커니즘을 연동한다. [3-1-3]

### 3.2. 마일스톤 및 산출물 규격 (tas:3-2) [3-2]
- **마일스톤 1: 기본 빌드 환경 구축 및 라이브러리 연동** (기간: 1주차)
    - CMake를 활용하여 GLFW 및 OpenGL 라이브러리 링크 환경을 구성하고 빈 창을 띄우는 소스코드를 구현한다. [3-2-1]
- **마일스톤 2: Dear ImGui 프레임워크 통합** (기간: 2주차)
    - Dear ImGui 소스코드 및 OpenGL/GLFW용 백엔드를 연동하고 기본적인 데모 윈도우가 동작하도록 설정한다. [3-2-2]
- **마일스톤 3: 고급 GUI 레이아웃 고도화 및 검증** (기간: 3주차)
    - Docking 및 Multi-viewports 설정을 제어하는 코드 구현을 완료하고, 다중 작업 공간의 사용자 인터페이스 검증을 수행한다. [3-2-3]
