# 011-tab-dialog: requirements

1. 통합 윈도우(Main Integration Window)를 구현한다. [1]
    - 통합 윈도우의 타이틀은 "dlg_integration"으로 지정한다. [1-1]
    - 통합 윈도우 내부에 탭 컨트롤(ImGui Tab Bar)을 배치한다. [1-2]
    - 탭 컨트롤은 통합 윈도우의 크기에 맞추어 가득 차도록(Stretch/Fit) 렌더링 영역을 구성한다. [1-3]

2. 탭 컨트롤 내에 위치할 두 개의 하위 윈도우(Sub Dialogs) 영역을 구현한다. [2]
    - 첫 번째 탭 아이템은 첫 번째 하위 윈도우(sub_dialog_1)를 나타낸다. [2-1]
        - 이 탭의 이름은 "sub_dialog_1"로 지정한다. [2-1-1]
        - 탭 내부에는 구별을 위해 "sub_dialog_1"이라는 텍스트가 표시되도록 한다. [2-1-2]
    - 두 번째 탭 아이템은 두 번째 하위 윈도우(sub_dialog_2)를 나타낸다. [2-2]
        - 이 탭의 이름은 "sub_dialog_2"로 지정한다. [2-2-1]
        - 탭 내부에는 구별을 위해 "sub_dialog_2"이라는 텍스트가 표시되도록 한다. [2-2-2]

3. 검증 방식은 빌드 후 실제 스크린샷 촬영 및 확인을 거친다. [3]
    - 소스코드를 빌드하고 애플리케이션을 직접 실행하여 UI 구조를 확인한다. [3-1]
    - 요구한 탭 컨트롤과 하위 창 구조가 올바르게 렌더링되는지 확인하고 스크린샷을 촬영하여 검증에 사용하며, 검증 완료 후 스크린샷 이미지 파일은 최종적으로 프로젝트 내에 저장되지 않도록 삭제한다. [3-2]

4. 다이얼로그 UI를 객체지향적 구조로 구현한다. [4]
    - 모든 다이얼로그의 공통 인터페이스 역할을 하는 추상 클래스(예: `BaseDialog`)를 정의하며, 모든 다이얼로그 클래스는 이를 상속받아야 한다. [4-1]
    - 메인 통합 다이얼로그(IntegrationDialog) 및 하위 다이얼로그(SubDialog1, SubDialog2)는 각각 독립된 클래스로 구현한다. [4-2]
    - `main.cpp`에서는 오직 `IntegrationDialog` 클래스의 인터페이스(예: `Draw()`)에만 직접 접근하여 화면에 그린다. [4-3]
    - `IntegrationDialog`가 내부에서 하위 다이얼로그 객체들을 직접 생성하고, 수명 및 렌더링 흐름을 관리한다. [4-4]
    - 기존에 구현되어 있던 데모 창 및 컨트롤 패널 UI는 제거하지 않고 그대로 병행하여 렌더링한다. [4-5]
    - 화면의 가려짐이나 겹침 현상을 방지하기 위해 기존의 제어 패널("Fantastic Fiesta Control Panel") 창은 초기 시작 시 접힌(Collapsed) 상태로 실행되도록 제어한다. [4-6]
