# 007-screenshot-dpi-fix: requirements

1. 기능 요구사항 [1]
    - 모니터에 DPI 배율(예: 125%, 150% 등)이 적용된 환경에서도 화면 전체가 잘리지 않고 온전하게 캡처되어야 한다. [1-1]
    - 기존 캡처 스크립트(`scripts/take_screenshots.ps1`)의 정상 기능(다중 모니터 지원, 지정한 파일명 접두사 및 인덱스 부여, 절대경로와 디바이스 이름 출력 등)은 그대로 유지되어야 한다. [1-2]

2. 비기능 요구사항 [2]
    - 캡처 시 이미지 해상도는 배율이 적용된 모니터의 물리 해상도(실제 픽셀 크기)와 일치하거나, 잘림 현상 없이 전체 화면이 포함되도록 저장되어야 한다. [2-1]
    - Windows API 호출 시 DPI 인지(DPI Awareness) 설정을 적절히 변경하여 캡처 좌표 및 크기가 왜곡되지 않도록 처리한다. [2-2]
    - 동일한 PowerShell 세션 내에서 스크립트를 여러 번 연속적으로 반복 실행하더라도, C# 타입(`WinAPI.DpiUtil`) 중복 로드로 인한 예외 및 에러가 발생하지 않아야 한다. [2-3]
