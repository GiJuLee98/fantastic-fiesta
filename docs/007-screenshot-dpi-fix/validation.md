# 007-screenshot-dpi-fix: validation

1. DPI 인지 함수 실행 검증 (tas:1-1, 3-1) [1]
    - `take_screenshots.ps1` 실행 시 C# DPI 설정 API의 동적 컴파일 및 호출 과정에서 예외가 발생하지 않고 성공적으로 수행되는지 검증한다. [1-1]

2. 스크립트 연속 반복 실행 검증 (tas:2-1) [2]
    - 동일한 PowerShell 창/세션에서 스크립트를 2회 이상 연속적으로 실행했을 때, '유형이 이미 존재합니다'와 같은 타입 중복 오류 없이 정상적으로 실행되는지 검증한다. [2-1]

3. 스크린샷 영역 및 해상도 검증 (tas:4-1) [3]
    - 배율(예: 125%, 150%)이 적용된 환경에서 캡처 스크립트를 실행했을 때, 저장된 이미지 파일의 너비와 높이가 물리 해상도에 도달하고 우하단 영역의 잘림이 없는지 이미지를 검증한다. [3-1]
    - 캡처된 결과 이미지의 하단 부분을 직접 분석하여 윈도우 작업 표시줄(Taskbar) 영역이 정상적으로 표시되는지 확인하여 전체 화면이 포함되었는지 입증한다. [3-2]
    - 생성된 테스트 결과 이미지 파일의 크기(가로x세로 해상도)가 가로/세로 방향에 관계없이 반드시 1920*1080 또는 1680*1050 규격 중 하나와 일치하는지 실제 파일 메타데이터 및 속성을 통해 검증한다. [3-3]

4. 출력 포맷 검증 (tas:4-2) [4]
    - 스크립트 종료 후 "Result: Success" 메시지와 함께 생성된 PNG 파일의 절대 경로와 모니터 장치 이름의 매핑 정보가 정확히 한 라인씩 출력되는지 확인한다. [4-1]
