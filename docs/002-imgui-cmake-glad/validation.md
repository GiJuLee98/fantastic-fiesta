# 002-imgui-cmake-glad: validation

1. 계획서 파일 존재 및 인코딩 검증 [1]
    - `docs/002-imgui-cmake-glad/imgui_external_setup_plan.md` 파일이 정상적으로 존재하며, 한글 깨짐이 없는 UTF-8 인코딩 형식인지 검증한다. (tas:1-1) [1-1]
    - 계획서 구조가 개요, 개발 도구 가이드, 의존성 획득 가이드, external 폴더 트리, 체크리스트 순서로 정확히 구성되어 있는지 구조적 정합성을 검증한다. (tas:1-2) [1-2]

2. 사전 개발 도구 가이드 내용 검증 [2]
    - Visual Studio 2022 워크로드("C++를 사용한 데스크톱 개발") 설치 안내 및 CMake PATH 등록 설정 경고 사항이 사용자가 누락 없이 이해할 수 있도록 명료하게 작성되었는지 검증한다. (tas:2-1, 2-2) [2-1]

3. 외부 의존성 획득 지침 가이드 내용 검증 [3]
    - GLFW Source Package 다운로드 관련 설명이 정상적으로 제공되는지 검증한다. (tas:3-1) [3-1]
    - GLAD 생성용 웹사이트의 옵션 값들(C/C++, OpenGL 3.3 Core Profile)이 표 형식으로 누락 없이 작성되었는지 검증한다. (tas:3-2) [3-2]
    - ImGui의 GitHub 리포지토리 및 `docking` 브랜치에 관한 획득 커맨드라인 지침과 핵심 소스 파일 및 GLFW/OpenGL3 백엔드 코드의 파일 이름 목록이 정확히 기재되었는지 교차 검증한다. (tas:3-3) [3-3]

4. external 트리 다이어그램 시각화 검증 [4]
    - 가이드에 작성된 트리 다이어그램이 실제 `external/` 디렉토리에 위치해야 하는 디렉토리와 파일 관계를 논리적으로 올바르게 표기하고 있는지 트리 뷰를 직접 검사하여 검증한다. (tas:4-1, 4-2) [4-1]

5. 체크리스트 실효성 및 유효성 검증 [5]
    - 환경 변수 확인을 위해 기재된 터미널 명령어(`cmake --version` 등)가 Windows PowerShell/CMD에서 올바르게 실행되는 실제 명령어인지 검증한다. (tas:5-1) [5-1]
    - 사용자가 파일 누락 여부를 판단하기 위한 자가진단 항목이 누락 파일 없이 체크 박스(`- [ ]`) 형태로 제공되어 체크가 가능한 상태인지 검증한다. (tas:5-2) [5-2]
