# Gemini Agent Skill 추가 가이드 [1]

- **버전**: v1.0 [1-1]
- **목적**: 이 문서는 Gemini Agent Skill의 확장 방법을 안내하는 가이드입니다. [1-2]
- **대상 독자**: 초심자를 포함하여 Gemini 에이전트의 기능을 추가 및 확장하고자 하는 모든 개발자 [1-3]

---

## 1. 개요
Gemini 에이전트는 다양한 Skill을 통해 기능을 확장할 수 있습니다. 본 가이드는 개발자가 새로운 Skill을 손쉽게 정의하고 에이전트에 추가할 수 있도록 돕기 위해 작성되었습니다.

---

## 2. Gemini Agent Skill의 개념과 필요성 [2-1]

- **개념**: Gemini Agent Skill은 스마트폰의 **'앱(App)'**과 같습니다. 스마트폰 자체에도 기본적인 기능(전화, 문자)이 내장되어 있지만, 배달 앱이나 지도 앱, 게임 앱 등을 설치하여 필요한 기능을 계속해서 확장해 나가는 것처럼, Gemini 에이전트 역시 새로운 Skill을 추가하여 파일 편집, 웹 검색, 코드 컴파일 등의 능력을 자유롭게 장착할 수 있습니다.
- **필요성**: 기본 AI 모델은 텍스트를 생성하고 분석하는 능력이 뛰어나지만, 실제 파일 시스템을 제어하거나 외부 도구를 사용하는 등 물리적인 작업을 직접 수행할 수는 없습니다. Skill은 AI 모델이 컴퓨터 시스템과 소통할 수 있는 통로가 되어, 더 정교하고 강력한 작업을 직접 처리할 수 있게 해줍니다.

---

## 3. 폴더 및 파일 구조 [2-2]

새로운 Skill을 에이전트에 성공적으로 등록하고 실행하기 위해서는 아래와 같은 표준화된 폴더 및 파일 구조를 준수해야 합니다.

```text
.gemini/
└── skills/
    └── [skill-name]/            # 추가할 Skill의 영문 소문자 이름 폴더
        ├── SKILL.md             # Skill의 명세, 기능 정의, 사용 가이드 문서
        ├── config.json          # Skill 동작 설정을 정의한 JSON 파일
        └── scripts/             # Skill 실행 시 필요한 스크립트 파일들을 배치하는 폴더
            └── run.ps1          # 예시: 실제 기능을 동작시키는 PowerShell 스크립트
```

- **`[skill-name]/`**: 새로 정의하는 Skill의 고유 식별자 폴더입니다.
- **`SKILL.md`**: Skill의 기능 개요, 세부 매개변수 구성 및 반환 결과 포맷 등을 문서화하는 필수 파일입니다.
- **`config.json`**: Skill 실행 환경, 필수 권한, 외부 의존성 패키지 등을 선언하는 설정 파일입니다.
- **`scripts/`**: Skill 내부에서 호출하여 작업을 처리할 실행 스크립트나 바이너리가 위치합니다.

## 4. 신규 Skill 추가 상세 절차 [3]

### 1단계: 폴더 생성 및 명명 규칙 [3-1]
새로운 Skill을 개발하려면 먼저 `.gemini/skills/` 디렉터리 내에 독립적인 개별 폴더를 생성해야 합니다.
- **명명 규칙**: 폴더명 및 Skill의 고유 식별자는 영문 소문자와 하이픈(`-`)만을 사용해야 합니다.
  - **올바른 예시**: `compile-project`, `image-generator`, `db-connector`
  - **잘못된 예시**: `compileProject` (대문자 사용), `compile_project` (언더바 사용), `compile project` (공백 포함)

### 2단계: SKILL.md 작성 요령 및 구성 요소 [3-2]
`SKILL.md`는 에이전트가 해당 Skill의 역할과 사용 방법, 지원하는 도구 목록 등을 인지하는 데 가장 중요한 문서입니다.
- **필수 구성 요소**:
  - **Name**: Skill의 직관적인 이름
  - **Description**: 에이전트가 이 Skill을 어떤 상황에 사용해야 하는지 상세하게 서술한 설명
  - **Instructions**: 에이전트가 Skill을 호출하고 실행할 때 지켜야 할 내부적인 가이드와 작업 흐름
  - **Tools**: 해당 Skill이 에이전트에 노출하는 세부 도구(함수)들의 인터페이스(매개변수 타입, 필수 여부, 반환값 구성 등) 명세
- **모범 사례 (Best Practices)**:
  - 매개변수의 설명은 AI가 오해하지 않도록 가능한 명확하고 세부적으로 기술하십시오.
  - 에이전트가 예외 발생 시(예: 실행 오류, 빈 값 입력 등) 대처해야 하는 구체적인 예외 처리 가이드를 작성하는 것이 안정성 향상에 좋습니다.

### 3단계: 외부 실행 스크립트 배치 및 통합 [3-3]
Skill이 실제로 컴퓨터 환경이나 프로젝트 내에서 어떠한 명령을 수행하려면 Python(`.py`), PowerShell(`.ps1`), Node.js(`.js`) 등의 외부 스크립트 파일이 필요합니다.
- **스크립트 배치**: 생성한 `[skill-name]/scripts/` 하위 폴더에 스크립트 파일을 한곳에 모아 배치합니다.
- **통합 및 연동**:
  - `SKILL.md` 상에서 에이전트가 `scripts/` 내의 특정 스크립트를 실행 명령어로 직접 구동할 수 있도록 도구(Tools)의 CommandLine 등에 상대 경로 형태로 명시하여 연결합니다.

### 4단계: 에이전트 설정을 통한 로드 및 활성화 [3-4]
작성한 Skill을 에이전트 시스템에 등록하여 최종적으로 동작시키기 위해서는 설정 파일을 편집해야 합니다.
- **설정 변경 대상**: `.gemini/config.json`
- **반영 방법**: `skills` 설정 항목의 리스트 내에 정의한 `[skill-name]`을 추가합니다.
  ```json
  {
    "enabled_skills": [
      "compile-project"
    ]
  }
  ```
- **활성화 확인**: 에이전트를 재구동하여 새로 추가한 Skill이 정상적으로 메모리에 로딩되고, 등록한 도구들이 활성화되는지 콘솔 출력이나 에이전트 상태를 통해 검증합니다.

## 5. 실전 예시 및 검증 가이드 [4]

### 실전 예시: 빌드 자동화 Skill (`build-helper`) [4-1]
다음은 프로젝트 빌드를 자동화하고 결과물을 검증하는 `build-helper`라는 이름의 가상 Skill 예시입니다.

**폴더 구조:**
```text
.gemini/
└── skills/
    └── build-helper/
        ├── SKILL.md
        ├── config.json
        └── scripts/
            └── build.ps1
```

**`SKILL.md` 전문:**
```markdown
# Build Helper Skill

- **Name**: Build Helper
- **Description**: 이 Skill은 프로젝트 소스 코드를 빌드하고 컴파일 성공 여부 및 결과 바이너리를 감지하는 빌드 자동화 도구입니다.
- **Instructions**:
  - 에이전트는 프로젝트 빌드를 수행하기 위해 반드시 `build_project` 도구를 호출해야 합니다.
  - 빌드 호출 전, 타겟 플랫폼(`platform` 매개변수)이 지정되었는지 확인하십시오. (기본값은 `win64`)
  - 빌드가 완료되면 결과 메시지를 분석하여 에러 로그가 존재하는지 확인하고, 성공 여부를 사용자에게 알려야 합니다.
- **Tools**:
  - `build_project`:
    - Description: 지정된 플랫폼 및 설정으로 프로젝트를 빌드합니다.
    - Parameters:
      - `platform` (string, optional): 빌드 타겟 플랫폼 (예: `win64`, `linux-x64`). 기본값은 `win64`.
      - `configuration` (string, optional): 빌드 모드 (`debug` 또는 `release`). 기본값은 `release`.
    - Returns:
      - `status` (string): 빌드 결과 상태 (`Success` 또는 `Failed`)
      - `log` (string): 빌드 시 발생한 표준 출력/에러 로그
```

### Skill 등록 검증 및 테스트 가이드 [4-2]

추가한 Skill이 정상적으로 동작하고 에이전트에 활성화되었는지 확인하려면 다음 절차를 수행해야 합니다.

#### 1. 에이전트 CLI를 통한 로드 확인
에이전트를 실행한 상태에서 다음 명령어 중 하나를 사용하여 활성화된 Skill 목록에 본인이 추가한 Skill이 포함되어 있는지 확인합니다.
- **CLI 명령어 예시**:
  ```bash
  /skills list
  ```
- **기대 결과**:
  ```text
  [Active Skills]
  - core-tools
  - build-helper (Loaded successfully from .gemini/skills/build-helper)
  ```

#### 2. 테스트 시나리오 검증
정상 작동 여부를 최종 검증하기 위해 시나리오 테스트를 수행합니다.

- **테스트 케이스 1: 기본 빌드 수행**
  - **입력 (사용자 발화)**: "프로젝트 릴리즈 빌드 돌려줘."
  - **에이전트 행동**: `build-helper` Skill의 `build_project` 도구를 플랫폼 `win64`, 구성 `release` 매개변수로 호출해야 함.
  - **통과 기준**: `Success` 상태와 함께 빌드 완료 메시지가 정상 출력됨.

- **테스트 케이스 2: 예외 매개변수 테스트**
  - **입력 (사용자 발화)**: "지원하지 않는 OS로 빌드해줘."
  - **에이전트 행동**: 매개변수 유효성 검증 규칙에 따라, 잘못된 플랫폼 값이 넘어갔을 때 스크립트 단에서 에러 코드를 반환하고, 에이전트가 예외 발생 상황을 요약하여 사용자에게 올바르게 피드백함.
