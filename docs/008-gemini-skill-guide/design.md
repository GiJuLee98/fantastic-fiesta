# 008-gemini-skill-guide: design

1. 문서 배치 및 포맷 정의 [1]
    - 안내서 문서는 `docs/008-gemini-skill-guide/gemini-skill-addition-guide.md` 경로에 마크다운 포맷으로 작성한다 (req:1) [1-1]
    - 처음 접하는 사람도 이해하기 쉬운 톤앤매너와 풍부한 시각적 요소(구조도 등)를 사용한다 (req:1-1) [1-2]
    - 안내서 문서 가장 상단에 문서 버전 정보(예: v1.0)를 명시한다 (req:1) [1-3]

2. gemini agent용 skill의 개념 및 필요성 기술 설계 [2]
    - agent skill이 무엇인지, 왜 필요한지 정의한다 (req:1-2) [2-1]
    - skill 추가 시 생성/수정해야 하는 프로젝트 폴더 및 파일(예: `.gemini/skills/` 폴더 구조, `SKILL.md` 및 부속 스크립트 등)의 명확한 구조적 정의를 명시한다 (req:1-2) [2-2]

3. 신규 skill 생성 및 추가를 위한 상세 절차 설계 [3]
    - 1단계: 신규 skill을 위한 디렉터리 구성 (req:1-3) [3-1]
    - 2단계: `SKILL.md` 작성 규칙(이름, 설명, 절차, 프롬프트, 도구 연동 등) 정의 (req:1-3) [3-2]
    - 3단계: skill에서 사용하는 실행 스크립트 또는 템플릿 파일 추가 방법 정의 (req:1-3) [3-3]
    - 4단계: 프로젝트 CLI 또는 환경 설정에 해당 skill을 등록하고 활성화하는 방법 정의 (req:1-3) [3-4]

4. 예시 및 검증 방법 설계 [4]
    - 구체적인 예시 skill(예: 빌드 자동화 skill 또는 테스트 실행 skill 등)의 전체 구조와 파일 내용을 예시로 제공한다 (req:1-4) [4-1]
    - 추가된 skill이 정상적으로 로드되고 에이전트에 의해 의도대로 동작하는지 확인하는 검증(Validation) 및 테스트 절차를 기술한다 (req:1-4) [4-2]
