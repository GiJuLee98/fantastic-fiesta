# 008-gemini-skill-guide: validation

1. 문서 생성 및 기본 메타데이터 검증 [1]
    - `docs/008-gemini-skill-guide/gemini-skill-addition-guide.md` 파일이 존재하고 정상적으로 읽을 수 있는지 확인한다 (tas:1-1) [1-1]
    - 문서 가장 상단에 버전 정보(v1.0)가 명시되어 있는지 확인한다 (tas:1-2) [1-2]
    - 문서의 도입부에 목적 및 대상 독자가 올바르게 설명되었는지 검토한다 (tas:1-3) [1-3]

2. 가독성 및 개념 정의 검증 [2]
    - 초심자 대상의 쉬운 설명과 비유가 포함되어 이해하기 쉬운 톤앤매너로 작성되었는지 검토한다 (tas:2-1) [2-1]
    - Skill 디렉터리 및 파일 구조(예: 트리 다이어그램)가 직관적이고 정확하게 표현되었는지 검토한다 (tas:2-2) [2-2]

3. 추가 절차 검증 [3]
    - Skill 폴더 생성 및 명명 규칙이 누락 없이 상세하게 기술되었는지 확인한다 (tas:3-1) [3-1]
    - `SKILL.md` 작성 방법(이름, 설명, 명령, 도구 등)이 명확하고 쉽게 따라 할 수 있게 정의되었는지 검토한다 (tas:3-2) [3-2]
    - 외부 실행 파일(스크립트 등)과의 통합 및 환경 설정 등록 방법이 오류 없이 실질적인 방법으로 안내되어 있는지 확인한다 (tas:3-3, 3-4) [3-3]

4. 실전 예시 및 검증 시나리오 검증 [4]
    - 제공된 예시 Skill 소스코드(또는 마크다운)가 문법 오류 없이 실제로 동작 가능한 형태인지 검토한다 (tas:4-1) [4-1]
    - 안내서에 기술된 검증 방법과 에이전트 CLI 명령어가 실제 사용 가능한 올바른 형식인지 대조하여 확인한다 (tas:4-2) [4-2]
