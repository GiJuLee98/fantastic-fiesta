# Gemini Agent Skill 추가 의사결정 템플릿

## 1. 개요 및 배경 [1-1]

- **배경**: 본 문서는 `docs/008-gemini-skill-guide/gemini-skill-addition-guide.md`에 정의된 Gemini Agent Skill 추가 가이드를 기반으로 작성되었습니다. 에이전트의 기능을 안전하고 체계적으로 확장하기 위해, 신규 Skill의 설계 단계에서 사전에 검토하고 의사결정해야 할 사항들을 명세하는 역할을 합니다. [1-1-1]
- **주요 사용 목적**: 새로운 Skill을 도입하기에 앞서, 해당 Skill의 필요성, 폴더 구조 설계, `SKILL.md` 필수 구성 요소, 외부 실행 스크립트와의 연동 방식, 그리고 에이전트 설정 파일 반영 계획을 표준화된 서식에 따라 구체화하고 의사결정을 기록하는 것을 목적으로 합니다. 이를 통해 구현 시 발생할 수 있는 식별자 중복, 매개변수 누락, 검증 실패 등의 예외 사항을 예방합니다. [1-1-2]

## 2. 활용 방법 [1-2]

- **템플릿 작성 순서**: [1-2-1]
  1. **요구사항 및 개요 정의**: Skill의 한글/영문 이름과 도입 목적을 명확히 정의합니다.
  2. **식별자 및 구조 설계**: 영문 소문자와 하이픈(`-`)만을 사용하는 명명 규칙을 적용하여 고유 식별자를 지정합니다.
  3. **SKILL.md 명세 구체화**: 에이전트가 이해할 명세(Name, Description, Instructions, Tools)를 설계합니다.
  4. **스크립트 및 설정 통합**: 연동할 명령어 실행 스크립트 및 `.gemini/config.json` 등록 방법을 확정합니다.
  5. **검증 시나리오 수립**: CLI 목록 확인 및 에이전트 테스트 시나리오를 구성합니다.
- **의사결정 승인**: 본 문서의 설계를 바탕으로 검토 및 승인이 완료된 후, 실제 디렉터리 생성 및 코드 구현(실제 Skill 파일 개발)을 진행합니다. [1-2-2]

## 7. 에이전트 환경 설정 및 활성화 결정 사항 [5]

- **`[5-1]` `.gemini/config.json` 등록 계획**:
  ```json
  {
    "enabled_skills": [
      "[skill-name]"
    ]
  }
  ```
- **`[5-2]` 로드 및 활성화 검증 계획**:
  - **CLI 로드 확인 명령어**: `/skills list`
  - **기대 출력 결과**: `[본인이 추가한 Skill 명이 Loaded successfully 와 함께 리스트업 되는지 검증]`
  - **동작 테스트 시나리오**:
    - **시나리오 1 (정상 동작)**: `[사용자 입력 -> 에이전트 도구 호출 -> 정상 응답 검증]`
