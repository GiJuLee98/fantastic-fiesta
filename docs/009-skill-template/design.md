# 009-skill-template: design

1. 템플릿 문서의 위치, 배경 명시 및 작성 형식 설계 (req:1-2, 1-4) [1]
    - 템플릿 파일의 경로는 `docs/009-skill-template/gemini-skill-template.md`로 설계한다. [1-1]
    - 템플릿 파일의 포맷은 Markdown(`.md`)으로 작성한다. [1-2]
    - 템플릿 문서 최상단에 `docs/008-gemini-skill-guide/gemini-skill-addition-guide.md` 가이드에 기반하여 새로운 Skill을 추가할 때의 의사결정을 지원하기 위해 생성된 템플릿이라는 배경 및 목적을 명시한다. [1-3]

2. 템플릿 문서의 세부 결정 사항 구조 및 숫자 인덱스 설계 (req:1-1, 1-3) [2]
    - 사용자가 에이전트와 협업할 때 각 항목을 정확히 가리키고 매핑할 수 있도록, 템플릿 내부의 모든 세부 결정 사항에 `[1-1]`, `[1-1-1]` 형태의 고유 숫자 인덱스를 명시한다. [2-1]
    - 템플릿의 장/절 및 세부 결정 항목 구조는 다음과 같이 설계한다. [2-2]
        - 1. 개요 및 배경 (배경 설명, 작성일, 작성자 등) [2-2-1]
        - 2. 폴더명 및 고유 식별자 결정 사항 (req:1-1-1) [2-2-2]
            - `[2-1]` Skill 폴더 및 고유 식별자 이름 (영문 소문자, 하이픈 구성 규칙 준수 여부)
        - 3. SKILL.md 세부 명세 결정 사항 (req:1-1-2) [2-2-3]
            - `[3-1]` Name (에이전트가 인지할 Skill의 직관적 명칭)
            - `[3-2]` Description (Skill의 구체적 용도와 에이전트 호출 조건)
            - `[3-3]` Instructions (에이전트가 Skill을 다룰 때 지켜야 할 내부 절차 및 예외 처리)
            - `[3-4]` Tools (에이전트에 노출할 도구들의 세부 명세)
                - `[3-4-1]` 도구명 및 설명
                - `[3-4-2]` 매개변수 구성 (타입, 필수 여부, 설명)
                - `[3-4-3]` 반환값 구성 (타입 및 데이터 예시)
        - 4. 외부 실행 스크립트 배치 및 통합 결정 사항 (req:1-1-3) [2-2-4]
            - `[4-1]` 실행 스크립트 파일 경로 (`scripts/[filename]`)
            - `[4-2]` 구현 언어 및 실행 환경 정의 (PowerShell, Python, Node.js 등)
            - `[4-3]` 스크립트 연동을 위한 CommandLine 실행 형식 및 매개변수 전달 규칙
        - 5. 에이전트 환경 설정 및 활성화 결정 사항 (req:1-1-4) [2-2-5]
            - `[5-1]` `.gemini/config.json`의 `enabled_skills` 내에 등록될 이름
            - `[5-2]` 에이전트 재구동 및 CLI 명령어 (`/skills list`) 실행을 통한 등록 검증 계획
